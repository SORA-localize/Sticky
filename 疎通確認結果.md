# 疎通確認結果

## 目的
- Tauri v2 + React + SQLite 方針で、`sticky` のMVPに必要な基礎機能が到達可能かを先に確認する
- 特に高リスクな `透過前面レイヤー` と `グローバルショートカット` を優先確認する

## 今回確認した項目
- グローバルショートカット登録
- 透過ウィンドウ設定
- 最前面表示
- フルスクリーン表示
- クリック透過切替 API の利用可否
- フロントエンドとのイベント疎通
- Rust / Cargo ビルド
- Tauri 開発起動

## 反映した内容

### Tauri 設定
- `app/src-tauri/tauri.conf.json`
  - `transparent: true`
  - `alwaysOnTop: true`
  - `decorations: false`
  - `fullscreen: true`
  - `visibleOnAllWorkspaces: true`
  - `skipTaskbar: true`
  - `acceptFirstMouse: true`
  - `macOSPrivateApi: true`

### Rust 側
- `@tauri-apps/plugin-global-shortcut` / `tauri-plugin-global-shortcut` を追加
- `Cmd + Shift + 1`
  - オーバーレイ表示/非表示切替
- `Cmd + Shift + 2`
  - クリック透過切替
- `set_ignore_cursor_events()` を使ったクリック透過切替を追加
- フロントに以下イベントを送出
  - `overlay://visibility`
  - `overlay://clickthrough`

### フロント側
- 疎通確認用の最小オーバーレイUIを追加
- 表示状態とクリック透過状態を確認できる簡易デバッグUIを追加

## 実行結果

### 通過
- `npm run build`
- `cargo check --manifest-path src-tauri/Cargo.toml`
- `npm run tauri dev`
  - `target/debug/sticky` の起動まで到達

## 確認できたこと
- Tauri v2 構成で、透過・最前面・フルスクリーン設定を持つウィンドウを起動できる
- グローバルショートカットプラグインを組み込み、ビルド・起動まで通る
- Rust 側からフロント側へイベント送出できる
- クリック透過切替 API をコード上で呼べる

## まだ手動確認が必要なこと
- メモカード以外の領域で、背面アプリを期待通り操作できるか
- `Cmd + Shift + 1` の表示/非表示切替が実際の macOS 上で安定するか
- `Cmd + Shift + 2` のクリック透過切替が期待通り効くか
- 他アプリのフルスクリーン表示中に、前面レイヤーがどう振る舞うか
- macOS の権限や環境差でショートカットが取りこぼれないか

## 現時点の判断
- 技術的に着手不能という状態ではない
- ただし、最重要リスクである `透過前面レイヤーの実操作感` はまだ手動確認が必要
- ここが不安定なら、アーキテクチャ再検討が必要になる可能性がある

## 次のアクション
