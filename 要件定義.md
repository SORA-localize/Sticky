# Post it 要件定義

## 1. 概要
本アプリは、Mac上で思考断片を即座に退避するための、セッション型・最前面表示・ポストイット風メモアプリである。  
目的は、作業中の思考の初速を落とさず、雑に書いて、後から管理画面で回収・整理できる体験を提供することにある。

## 2. プロダクト目的
- 起動から記入開始までを極限まで短くする
- 思考中に保存先や整理方法を考えさせない
- 複数の思考枝をセッション単位で保持できるようにする
- 書いた内容は失わず、後から管理画面で確認・移動・削除できるようにする
- 一般的なMacアプリに近い自然な挙動を保ちつつ、メモUIは極力軽くする

## 3. 対象プラットフォーム
- 初期MVP対象: `macOS`
- 初期実装技術方針: `Tauri v2 + React + SQLite`
- iPhone / Windows / Android / 同期機能は初期MVP対象外とする

## 4. コアコンセプト
- メモは「整理」ではなく「退避」のために使う
- メモは毎回新規作成感を持つ
- セッションは思考のまとまりを表す論理グループである
- デスクトップ上では、他アプリより前面に出る
- 邪魔になったらすぐ閉じられ、内容は後から回収できる

## 5. 用語定義
- `Memo`: 個別のメモ。本文、タイトル、位置、状態を持つ最小単位
- `Session`: 複数メモを束ねる論理グループ。1枚セッションまたは複数枚セッションとして生成される
- `管理画面`: 保存済みメモ・セッションを一覧、検索、移動、削除、復元する画面
- `表示中`: デスクトップ上にメモが出ている状態
- `閉じる`: デスクトップ上から非表示にするが削除はしない
- `削除`: ゴミ箱へ移す操作
- `編集`: メモ本文にカーソルを置き入力可能な状態
- `セッション選択`: セッション内メモ全体が選択対象になっている状態
- `メモ選択`: セッション内の個別メモのみが選択対象になっている状態

## 6. スコープ
### MVPに含めるもの
- 1枚セッション・複数枚セッションの生成
- 最前面メモ表示
- メモ入力、保存、閉じる、削除
- セッション単位管理
- 管理画面
- ゴミ箱
- 簡易検索
- 自動保存
- 自動閉じる
- 15スロットの初期配置

### MVPに含めないもの
- クラウド同期
- 他OS対応
- アプリ/ウィンドウ文脈連動メモ
- AI整理
- 高度なタグ/フォルダ管理
- 共有機能
- 画像添付
- Markdownの高度編集機能

## 7. 機能要件
### 7.1 セッション作成
- ユーザーは `1枚セッション` または `複数枚セッション(2〜9枚)` を新規作成できる
- セッション生成時に、内部的に `Session > Memo` 構造を作成する
- セッションは生成後にメモ追加できない
- 1枚セッションで追加作成した場合は別セッションとする
- 複数枚セッションで不要メモを削除して1枚になっても、その後の新規メモは別セッションとする

### 7.2 表示上限
- 同時展開可能セッション数は最大 `5`
- デスクトップ上の同時表示メモ数は最大 `15`
- 上限到達時は新規作成を行わない
- 上限到達時は、現在表示中のメモ群全体に対し、赤系半透明オーバーレイと横揺れ演出を出す
- 管理画面には上限状態を明示する
  - `セッション上限: 5/5`
  - `表示メモ上限: 15/15`

### 7.3 メモ表示
- メモは他アプリより前面に表示する
- メモ表示時はOS標準のウィンドウ装飾を出さない
- メモはポストイット風のUIとする
- セッションごとに固定色を持つ
  - 1: 赤
  - 2: 橙
  - 3: 黄
  - 4: 緑
  - 5: 青
- セッション作成時は、現在デスクトップ上で開いているセッションが使用していない最小 `color_slot` を自動割り当てする
- 閉じたセッションの色スロットは解放される
- 色味は目に優しい低刺激なトーンにする

### 7.4 初期配置
- デスクトップ上の初期表示位置は `15スロット` で管理する
- スロットは `5列 x 3段` の固定構成とする
- スロット座標は画面サイズに対する相対座標で持つ
- 基本座標は以下とする
  - 横: `13%, 30%, 50%, 70%, 87%`
  - 縦: `18%, 43%, 68%`
- 新規表示時は空いている最小番号スロットへ配置する
- ユーザーがメモを移動した場合、その元スロットは空きとして再利用可能とする
- 新規メモは常に空いている最小番号スロットから前詰めで配置する
- 再表示時も同様に空きスロットへ配置する
- 将来的に位置復元を検討可能だが、MVPでは固定スロット優先とする

### 7.5 選択・編集
- シングルクリックで `メモ選択`
- ダブルクリックまたは `Enter` で `編集状態` に入る
- 右クリックメニューの `このセッションを選択` で `セッション選択` に入る
- `Esc` で編集終了または選択解除する
- セッション選択中はセッション内全メモを一括移動できる
- メモ選択中は対象メモのみ移動できる
- クリック位置から `6px` 以上移動した時点でドラッグ開始とする

### 7.6 保存・閉じる・削除
- `Cmd + S`: 選択対象を更新保存する
- `Cmd + Enter`: 保存して閉じる
- `Delete`: 削除確認モーダルを開く
- 削除確認中の `Delete` または `Enter`: 削除確定
- 削除確認中の `Esc`: キャンセル
- 削除は即時物理削除ではなくゴミ箱へ移動する
- デスクトップ上の右クリックメニューには以下を表示する
  - `このセッションを選択`
  - 区切り線
  - `このセッションを削除...`
  - `このセッションを閉じる`

### 7.7 自動保存・自動閉じる
- 内部自動保存は `5分ごと` に実行する
- 自動保存は、デスクトップ上に開いているセッションが存在する時だけ実行する
- 対象は未保存変更を持つメモのみとする
- 保存処理には明示保存と自動保存の両方を含み、そのたびに `title` を本文先頭10文字から再生成する
- 自動閉じる時間は設定で変更可能とする
  - `5分 / 10分 / 30分 / 60分`
- 自動閉じる判定はセッション単位とし、各セッションの `last_active_at` を基準とする
- 選択中・編集中の対象は自動閉じる対象外とする
- 自動閉じる時は保存したうえでデスクトップから非表示にする
- 自動シャットダウン機能は持たない

### 7.8 管理画面
- 管理画面は `Home / ゴミ箱 / 設定` の3タブ構成とする
- Homeは日時降順で表示する
- 表示構造は `日時 > セッション > メモ`
- メモはポストイット風カードで表示する
- 各メモには簡易タイトルを表示する
- タイトルは本文先頭 `10文字程度` を保存処理時に自動採用する
- 管理画面には `content != ''` のメモのみ表示する

### 7.9 管理画面での操作
- 右クリックで削除メニューを表示する
- メモ選択後に `Delete` で削除確認を行える
- メモはドラッグ&ドロップで別セッションへ移動できる
- セッション移動時は色だけでなく、`session_id` を移行先へ変更する
- 移動後は移行先セッション色を適用する
- 空になったセッションは残さない

### 7.10 ゴミ箱
- 削除対象はゴミ箱へ移動する
- ゴミ箱保持期間は `30日`
- ゴミ箱内で復元、完全削除が可能
- 30日経過後は自動削除する

### 7.11 検索
- 初期検索はキーワード一致のみ
- 検索対象はメモ本文およびタイトル
- 結果は日時降順で表示する

## 8. ショートカット要件
### 必須ショートカット
- `Cmd + Option + Enter`: 1枚セッション開始
- `Cmd + Option + N`: 新規セッション枚数選択UIを開く
- 枚数選択UI内の `1` 〜 `9`: 指定枚数セッション開始
- `Cmd + S`: 保存
- `Cmd + Enter`: 保存して閉じる
- `Delete`: 削除確認
- `Esc`: 選択解除 / 編集終了 / モーダルキャンセル
- `Enter`: 編集開始または編集中改行

### 将来検討ショートカット
- 管理画面を開く
- ゴミ箱を開く
- 設定を開く
- セッション全選択
- 次/前セッションフォーカス移動

## 9. データ要件
### 最低限保持するデータ
- Session
  - `id`
  - `created_at`
  - `updated_at`
  - `color_slot`
  - `is_open`
  - `last_active_at`
  - `trashed_at`
- Memo
  - `id`
  - `session_id`
  - `content`
  - `title`
  - `created_at`
  - `updated_at`
  - `last_edited_at`
  - `is_open`
  - `is_dirty`
  - `trashed_at`
- Settings
  - `auto_close_minutes`
  - `max_open_sessions`
  - `max_open_memos`

## 10. セッション0件時の扱い
- あるセッション内のメモ数が `0` になった場合、そのセッションは自動的に消滅する
- 表示中なら閉じる
- 管理画面からも消す
- 空セッションは保持しない

## 10.1 起動時クリーンアップ
- アプリ起動時は、まず全 `sessions.is_open` と `memos.is_open` を `0` にリセットする
- その後、`content = ''` かつ `updated_at = created_at` の空メモを削除する
- 空メモ削除後、メモを持たないセッションを削除する
- これにより、クラッシュ復帰時の上限判定・色スロット解放・空データ残留を防ぐ

## 11. 非機能要件
- 起動から入力開始までが速いこと
- 入力操作に対して遅延を感じにくいこと
- 常駐時の負荷が低いこと
- 一般的なMacアプリに近いフォーカス・ドラッグ・重なり挙動を保つこと
- 特殊すぎる独自UIより、既存アプリに近い直感的挙動を優先すること

## 12. UI要件
- デスクトップ上のメモUIはクロームレス
- 管理画面は通常アプリUIでよい
- 選択中メモは青枠表示
- セッション選択時はセッション内メモ全体を青枠表示
- 非選択状態は少し沈んだ見え方でもよい
- 編集中はカーソル表示と強めのフォーカス表現を持つ

## 13. 警告文言要件
### セッション上限
- `これ以上セッションは開けません`
- `展開中セッションは最大5件です`
- `新しいセッションを開くには、既存のセッションを閉じてください`

### メモ上限
- `これ以上メモは表示できません`
- `同時表示できるメモは最大15件です`
- `新しいメモを表示するには、既存のメモを閉じてください`
